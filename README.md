## 项目简介

**履迹 Resume Trail** 是一个围绕求职全过程设计的管理产品。  
它不是简单的岗位记录表，也不是把 Excel 换成更好看的界面。  
它更像一套为个体设计的求职工作台：在每一次投递、每一轮面试、每一份简历版本和每一次复盘之间，建立起清晰、连续、可追踪的关系。

比起“多记一点”，履迹更在意的是另一件事：  
**让求职这件本来容易失控的事，重新变得有节奏、有判断，也有积累。**

项目从真实使用场景出发，重点解决这些问题：

- 投递信息分散，更新一轮状态后很容易失控
- 简历版本越来越多，但不知道哪一份更适合什么方向
- 面试结束后有复盘冲动，但很难沉淀成可复用资产
- 手机、电脑来回切换时，数据不一致、体验断裂
- 记录很多，却很难判断当前求职策略到底有没有变好

当前正式站点：

- [resumetrail.site](https://resumetrail.site/)

## 为什么是履迹

很多工具都可以记录求职，但很少有工具真正理解求职。

履迹想解决的，不是“你能不能把岗位填进去”，而是当投递越来越多、信息越来越碎、节奏越来越乱时，是否还有一套界面和逻辑，能够帮你重新看清：

- 现在最该推进的是哪几个机会
- 哪一份简历更适合当前方向
- 每一次面试后的经验，能不能变成下一次的准备
- 记录下来的这些数据，究竟有没有帮助你做出更好的判断

它的价值不在于把求职做成更复杂的系统，而在于把原本分散在表格、文档、备忘录和脑海里的内容，重新组织成一个持续可用、能沉淀判断的个人工作台。

如果说很多工具擅长“帮你记住”，  
那么履迹更在意的是：**帮你推进，帮你复盘，也帮你慢慢建立自己的节奏。**

## 核心能力

### 1. 求职看板

将岗位按流程阶段组织在同一条工作流里，支持从观望、投递、笔试、面试到 Offer / 流程终止的持续推进。

- 支持横向看板管理
- 支持状态更新与时间线联动
- 支持岗位详情、偏好度、Base 地、渠道链接、绑定简历等信息

### 2. 表格与快捷编辑

除了看板视角，也提供更适合批量整理的表格视角，便于做筛选、排序、字段管理和快速修订。

- 自定义列
- 字段筛选与排序
- 快捷编辑
- 分类与标签体系

### 3. 简历文件仓

把不同版本的简历作为长期资产管理，而不是临时附件。

- 上传 / 维护多份简历
- 为简历补充标签、备注和适用记录
- 与投递岗位建立关联
- 追踪某份简历对应的投递、推进和面试表现

### 4. 复盘沉淀

把每一次面试后的记录从“写完就丢”变成下一次还能调用的经验资产。

- 结构化复盘
- 按轮次、岗位、标签归档
- 支持录音转文字后的复盘整理
- 支持后续复盘删除与维护

### 5. 日历与时间节奏

把投递截止、面试安排、准备动作与时间轴结合起来，帮助用户管理求职节奏，而不是只管理结果。

- 月 / 周视图
- 当天安排与即将到来提醒
- 面试、DDL、跟进事项统一展示
- 移动端单独优化

### 6. 数据分析

让记录真正变成判断依据，而不是停留在“我记了很多”。

- 投递趋势
- Offer 转化漏斗
- 岗位类别转化率
- 渠道表现
- 城市分布
- 数据解读

### 7. 登录与云端同步

支持体验模式和邮箱模式两条路径。

- 体验模式：本地使用、快速上手
- 邮箱验证码注册：首次验证后可直接密码登录
- 刷新后保持登录态
- 同账号多端同步
- 主题偏好同步到云端

### 8. 主题系统

提供深色 / 浅色双主题，不是简单换底色，而是完整覆盖登录页与站内界面的统一主题系统。

- 登录页与站内联动切换
- 浅色模式为暖调玻璃质感
- 深色模式保持沉浸感与高对比
- 背景粒子交互随主题切换

## 产品结构

当前产品主要由 6 个核心页面组成：

1. **看板**：按阶段管理岗位推进
2. **表格**：做批量整理、筛选、排序和快捷编辑
3. **简历**：管理简历版本、备注与适用记录
4. **复盘**：沉淀面试记录与问题标签
5. **日历**：查看面试、DDL 与节奏安排
6. **数据**：查看趋势、漏斗、渠道和类别表现

辅助模块包括：

- 登录 / 注册 / 忘记密码
- 我的资料
- 分类与标签管理
- 数据管理与清空重置
- 浅色 / 深色主题切换
- PostHog 埋点分析

### 使用方式

#### 体验模式

- 无需注册
- 可直接进入产品体验核心功能
- 数据保存在当前设备

#### 邮箱模式

- 首次输入邮箱和密码
- 收取验证码完成注册
- 后续直接邮箱 + 密码登录
- 数据自动同步到 Supabase 云端

## 项目特点

### 场景真实

这个产品不是从抽象的“求职管理”概念开始做的，而是从真实使用过程中的混乱、遗忘、重复整理和策略失焦开始一点一点长出来的。

### 体验驱动

大量细节迭代都围绕“长期使用是否舒服”展开，包括：

- 登录体验
- 卡片布局
- 简历仓交互
- 日历可读性
- 数据页层级
- 浅色 / 深色主题的一致性

### 云端真源

业务数据不再依赖 localStorage 作为主存，而是以 Supabase 为云端真源，支持跨设备同步与刷新恢复。

### PM 导向

它不只是一个能跑的前端项目，也是一套可以体现产品经理全流程思考的作品：

- 场景洞察
- 功能定义
- 信息架构
- 交互设计
- 迭代打磨
- 数据埋点
